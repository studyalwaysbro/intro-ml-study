Module 1: Core Concepts

Mitchell definition

A program learns from experience E with respect to task T and performance measure P if its performance at T, as measured by P, improves with E.
  • `T`, `E`, and `P` are required in the definition.
  • A particular model `H` is not required by the definition itself.
  • Supervised learning uses labeled data; unsupervised learning uses unlabeled data.
  • Online learning updates incrementally; batch learning trains on the full available set.
  • Model-based learning fits parameters; instance-based learning compares to stored examples.
Most likely trap: �Which item is not required in the definition of machine learning?� The answer is usually `Model H`.

Module 2: Linear Regression

Core model

y-hat = w0 + w1x1 + ... + wnxn

Key formulas

MSE = (1/m) * Sum[(y-hat(i) - y(i))^2] Normal equation: w = (X^T X)^(-1) X^T y Single-sample GD update: wj <- wj - alpha * 2(y-hat - y)xj
  • Linear regression has a closed-form solution; logistic regression does not.
  • If learning rate is too large, GD may oscillate or diverge.
  • Lasso can drive coefficients to zero; Ridge usually shrinks them without zeroing them.
  • MAE is more robust to outliers than MSE/RMSE.
Most likely trap: more training iterations do not automatically solve overfitting.

Module 3: Logistic Regression

P(y=1|x) = sigma(w^T x) sigma(x) = 1 / (1 + e^(-x)) sigma'(x) = sigma(x)(1 - sigma(x))
  • Binary classification model that predicts a probability-like output.
  • Standard loss is cross-entropy / log loss.
  • With cross-entropy, the optimization is convex in the standard setup.
  • No normal-equation-style closed-form solution.

Module 3: Evaluation

Precision = TP / (TP + FP) Recall = TP / (TP + FN) F1 = 2TP / (2TP + FP + FN) TPR = TP / (TP + FN) FPR = FP / (FP + TN)
  • ROC plots TPR vs FPR.
  • ROC close to the diagonal means weak or random-like performance.
  • Accuracy can be misleading on imbalanced datasets.
  • One-vs-rest is a standard multiclass extension.

Module 4: Decision Trees

  • Trees are hierarchical rule structures.
  • Classification tree: discrete targets. Regression tree: continuous targets.
  • Training is greedy.
  • ID3 uses information gain.
  • CART uses Gini impurity.
Entropy(S) = - Sum[p_i log2 p_i] IG(S,A) = Entropy(S) - Sum_v (|S_v|/|S|) Entropy(S_v) Gini = 1 - Sum[p_k^2]
  • Reduced error pruning uses validation performance.
  • Chi-squared pruning tests whether a split is statistically meaningful.
Most likely trap: decision trees are not only for classification. They are also used for regression.

Module 5: Bias, Variance, Cross-Validation

Expected test error � Bias^2 + Variance + irreducible noise
  • High bias means underfitting.
  • High variance means overfitting.
  • K-fold cross-validation is used for model and hyperparameter selection.
  • After tuning, retrain on all training data and test once on the held-out test set.

Module 5: Ensembles

  • Bagging: sampling with replacement.
  • Pasting: sampling without replacement.
  • Boosting: sequentially corrects earlier mistakes.
  • Stacking: uses a separate learned model to combine base learners.
  • Bagging helps reduce variance for unstable learners like trees.

Module 6: SVM

  • SVM finds an optimal separating hyperplane with maximum margin.
  • Support vectors are the points nearest the boundary.
  • Kernel functions allow nonlinear separation.
  • The lecture emphasizes quadratic optimization and the avoidance of poor local minima.
  • SVR is the regression version of SVM.

Module 7: Bayesian Learning

P(A|B) = P(B|A)P(A) / P(B)
  • If P(A|B) = P(A), then A and B are independent and P(AB) = P(A)P(B).
  • MAP picks the hypothesis with highest posterior probability.
  • Bayes optimal classifier uses all hypotheses weighted by posterior probability.
  • Naive Bayes assumes conditional independence given the class.
  • Gibbs is worth memorizing as a separate algorithm with its expected-error guarantee.

Module 8: Neural Networks

  • Perceptron is the classic single-layer neural model.
  • Backpropagation is the standard multilayer training method.
  • Sigmoid is a smooth differentiable threshold-like activation.
  • Binary classification often uses sigmoid; multiclass often uses softmax.
  • Regression and classification networks usually use different loss functions.
  • A single-layer perceptron cannot solve XOR.
g(x) = 1 / (1 + e^(-x)) g'(x) = g(x)(1 - g(x))
